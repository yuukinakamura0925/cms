## 📝 ドキュメント作成ガイド

### 命名規則
- ファイル名は小文字とハイフンを使用: `response-rules.md`
- ディレクトリ名も小文字とハイフンを使用: `api/`, `architecture/`

### フォーマット
- Markdown形式で記述
- 日本語で記述
- コード例は適切なシンタックスハイライトを使用

### 更新ルール
- 機能追加・変更時は関連ドキュメントを更新
- ドキュメントの更新履歴を記録
- レビュー後にマージ
